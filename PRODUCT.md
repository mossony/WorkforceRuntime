# Product

## Register

product

## Users

Workforce Runtime is used by builders and operators who coordinate AI worker organizations. They are trying to design an org, launch a task, observe live execution, inspect individual agents, steer work when needed, and review final reports without reading raw logs first.

## Product Purpose

The product is an organizational control plane for third-party AI workers such as Codex, Claude Code, and OpenRouter-hosted agents. It should make organization structure, task state, agent output, budgets, tools, traces, and human-facing reports understandable while preserving access to detailed debugging evidence.

## Brand Personality

Calm, operational, precise. The interface should feel like a serious control room for human-AI work rather than a marketing page or a decorative agent demo.

## Anti-references

Avoid marketing hero pages, decorative SaaS gradients, nested card dashboards, side-stripe alert styling, and cluttered walls of debug output as the default experience. Debug detail is still valuable, but it should be one deliberate mode away from the primary workflow.

## Design Principles

- Make the next operational action obvious: describe the task, design the org, run the work, inspect the result.
- Show structure before noise: the org tree and current agent summaries should be readable at a glance.
- Keep evidence close: every simplified state needs a path to streams, tools, prompts, and trace details.
- Preserve operator control: configuration, steering, and debug mode remain available without dominating the default screen.
- Prefer restrained product UI patterns over expressive decoration.

## Accessibility & Inclusion

Target WCAG AA contrast, keyboard-accessible controls, readable summaries, clear focus states, and reduced-motion behavior for non-essential transitions.
